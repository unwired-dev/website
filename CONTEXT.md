# Unwired Website

This context defines the business language for Unwired's public company website. It exists to keep positioning, product references, and service language consistent as the site is designed and built.

## Language

**Unwired**:
The company behind the website, offering React and frontend consultancy while also building its own software products.
_Avoid_: Agency, studio, vendor

**Unwired Mail**:
A privacy-focused email client for macOS and iOS that uses local AI, presented on the company website as proof of product capability.
_Avoid_: Side project, demo app

**Company Website**:
The public marketing site for Unwired, whose primary purpose is to generate consultancy leads and whose secondary purpose is to establish credibility for the company's product work.
_Avoid_: Product site, docs site

**Consultancy Lead**:
A prospective client engagement for React or frontend consulting services initiated through the website.
_Avoid_: User signup, app install

**Primary Buyer**:
The CTO, engineering manager, or startup founder evaluating Unwired for senior React and frontend consulting on product delivery, UI architecture, or design-system work.
_Avoid_: End user, app customer, subscriber

**Primary Offer**:
Unwired helps product teams ship better React frontends through senior implementation, frontend architecture, design-system work, and targeted advisory.
_Avoid_: General software consulting, full-service development

**Primary Positioning**:
Unwired acts as a temporary frontend lead for product teams, most often through clearly scoped project engagements.
_Avoid_: Staff augmentation, outsourced dev shop

**Founder Proof**:
The primary source of credibility for the website is Jan Silhan's professional track record, with LinkedIn serving as the public record of relevant experience and work history.
_Avoid_: Anonymous company proof, purely brand-led credibility

**Founder-Led Consultancy**:
Unwired is presented as a consultancy brand led directly by its founder, with the founder's expertise and judgment central to the buying decision.
_Avoid_: Agency collective, company-first positioning

**Homepage Promise**:
Unwired helps product teams move faster on React frontend work with senior hands-on leadership.
_Avoid_: Full-service product studio messaging, generic innovation claims

**Product Proof Page**:
Unwired Mail should have its own dedicated page on the company website, while remaining subordinate to the site's consultancy-first narrative.
_Avoid_: Product-only landing site, equal-weight site sections

**Growth-Ready Static Site**:
The website should launch as a small static marketing site, while using a stack that can expand into articles and case studies later without requiring a rewrite.
_Avoid_: Throwaway brochure site, CMS-first platform

**Repo-Owned Content**:
The website's content should live in the repository as Markdown or MDX and be edited directly in code for the first version.
_Avoid_: CMS-managed content, database-backed editorial workflow

**React-Based Static Framework**:
The website should be built with a React-based framework so the implementation itself reflects the frontend expertise Unwired is selling, while still deploying as a static site.
_Avoid_: Plain HTML brochure stack, non-React primary framework
